# Product Manager Agent — Handover Contracts

## Purpose

This document defines what the Product Manager Agent receives, produces, and hands off to other agents and the Human Decision Approver.

## Inputs Received

- Approved Requirement Specification
- Requirement-to-Product Handover
- Open Human Decisions from requirements phase
- Business constraints from Human Decision Approver

## Outputs Produced

- Product Plan document
- MVP scope definition
- Feature priority matrix
- Release plan
- Success metrics
- GitHub feature issue list

## Handover Obligations

- UX/UI Agent: Product Plan and Product-to-UX Handover
- Enterprise Architect Agent: Product-to-Architecture Handover
- Human Decision Approver: Product Plan for Gate 2 approval

## Approval Gates

All deliverables must pass the relevant approval gate before the next agent activates. See [Approval Gates Overview](../../04-approval-gates/approval-gates-overview.md).

## GitHub Commit Requirements

No handover is official until committed to GitHub via pull request. Agent chat output is temporary until committed.
