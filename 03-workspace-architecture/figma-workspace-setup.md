# Figma Workspace Setup

## Principle

Figma is the design workspace. GitHub stores the Figma links, approval notes, UX rationale, sample data, and design handover.

## One Figma Project Per Solution

Each software solution gets its own Figma project with a standardized page structure.

## Standard Pages

```text
00 Cover
01 Personas
02 User Journey
03 Information Architecture
04 Wireframes
05 Clickable Prototype
06 UI Design
07 Components
08 Design System
09 Developer Handoff
```

## Page Purposes

| Page | Content |
|---|---|
| 00 Cover | Project name, version, links to GitHub |
| 01 Personas | User personas with roles and needs |
| 02 User Journey | End-to-end user journey maps |
| 03 Information Architecture | Site map, navigation structure |
| 04 Wireframes | Low-fidelity wireframes (Mode 1 UX) |
| 05 Clickable Prototype | Interactive prototype for human testing |
| 06 UI Design | High-fidelity screens (Mode 2 UI) |
| 07 Components | Reusable UI components |
| 08 Design System | Typography, colors, spacing standards |
| 09 Developer Handoff | Specs, measurements, export notes |

## GitHub Integration

All Figma links must be documented in GitHub (`docs/04-ux-ui/figma-links.md`). Approval records and UX rationale live in GitHub, not Figma comments alone.
