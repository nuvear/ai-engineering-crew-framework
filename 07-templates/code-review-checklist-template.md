# Code Review Checklist

File path: docs/08-review/CODE-REVIEW-CHECKLIST-[work-item-id].md

## 1. Review Context

## 2. Traceability Checks

- [ ] Pull request links to approved requirement IDs
- [ ] Code changes map to implementation tasks
- [ ] Test evidence maps to requirement IDs
- [ ] Governance notes are included

## 3. Scope Checks

- [ ] No unapproved scope added
- [ ] Product scope boundaries followed
- [ ] UX/UI handover followed when applicable
- [ ] Known limitations documented

## 4. Architecture Checks

- [ ] Approved architecture followed
- [ ] Technical boundaries followed
- [ ] Integration assumptions documented
- [ ] Data model assumptions documented

## 5. Code Quality Checks

- [ ] Code is readable and maintainable
- [ ] Naming is clear
- [ ] Duplication is justified or avoided
- [ ] Error handling is appropriate
- [ ] Logging and observability are appropriate for the scope

## 6. Risk Checks

- [ ] Security concerns reviewed
- [ ] Privacy concerns reviewed
- [ ] Compliance concerns reviewed
- [ ] Sensitive data use is approved or absent
- [ ] Reliability risks are documented

## 7. Test Evidence Checks

- [ ] Tests were added or updated where appropriate
- [ ] Test commands or evidence are documented
- [ ] Untested areas are explained
- [ ] Material gaps block QA handover
